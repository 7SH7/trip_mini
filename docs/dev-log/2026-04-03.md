# 개발 일지 — 2026-04-03 (Day 1)

## 프로젝트 개요

**Trip** — 여행 플랫폼 MSA 프로젝트

취준생 포트폴리오용 여행 서비스. 숙소 조회, 여행 예약/결제, 실시간 채팅, 피드(사진/글), 실시간 스트리밍, 구독제 결제를 MSA + DDD 아키텍처로 구현한다.

### 기술 스택
- **Backend**: Kotlin + Spring Boot 4.0.5 (코프링)
- **DB**: MySQL + JPA (ddl-auto: update), QueryDSL + MyBatis 설정 완료
- **캐시/토큰**: Redis
- **메시징**: Kafka + Outbox Pattern
- **서비스 간 통신**: gRPC (동기), Kafka (비동기)
- **인증**: JWT + OAuth2 (Google/Kakao Authorization Code 방식)
- **보안**: XSS 필터, 보안 헤더, AES-GCM 내부 인증, BCrypt 해싱
- **인프라**: Eureka + Spring Cloud Gateway, Docker Compose (MySQL, Redis, Kafka, MinIO, Nginx-RTMP)
- **Frontend**: React + TypeScript + Vite, Redux Toolkit, React Query, MUI, Styled Components
- **빌드**: Gradle Kotlin DSL, 멀티모듈 모노레포

## 오늘 한 작업

### 1. 프로젝트 초기 셋업
- Spring Boot 단일 프로젝트 → 멀티모듈 MSA 구조로 전환
- Java 21 toolchain 설정
- 7개 모듈 생성: eureka-server, api-gateway, common, user-service, trip-service, booking-service, payment-service

### 2. DDD 패키지 구조 적용
- 각 서비스에 `domain/entity`, `domain/repository`, `application/service`, `application/dto`, `interfaces/controller` 계층 구성
- 기본 CRUD API 구현 (User, Trip, Booking, Payment)

### 3. Eureka + API Gateway
- Eureka Server (:8761) — Service Discovery
- API Gateway (:8080) — 모든 /api/** 요청 라우팅
- 각 서비스에 Eureka Client 등록

### 4. JWT 인증/인가
- user-service에 로그인/회원가입/토큰 갱신 구현
- Gateway에서 JWT 검증 필터 (GlobalFilter)
- X-User-Id, X-User-Email, X-User-Role 헤더 전달
- Redis로 refresh token 저장 + access token 블랙리스트 (로그아웃)

### 5. OAuth2 소셜 로그인
- 처음: redirect 방식 → access token 직접 전달 방식 → **Authorization Code 방식으로 최종 전환**
- 프론트가 authorization code를 백엔드에 전달 → 백엔드가 Google/Kakao에 token 교환 → userinfo 조회 → JWT 발급
- 구글 access token이 프론트에 노출되지 않는 안전한 구조

### 6. 내부 서비스 인증 (AES-GCM)
- 서비스 간 통신 시 X-Internal-Token 헤더로 인증
- AES-GCM 대칭키 암호화 (30초 만료)
- InternalAuthFilter + InternalRestClientConfig

### 7. 예외 처리 + Validation (Phase 1A, 1B)
- common 모듈에 BusinessException 계층 (EntityNotFoundException, InvalidRequestException, ServiceUnavailableException)
- GlobalExceptionHandler (@RestControllerAdvice)
- 모든 Request DTO에 @NotBlank, @NotNull, @Email, @Positive 적용
- 모든 Controller에 @Valid 적용

### 8. Kafka 이벤트 구조화 (Phase 1C)
- String 방식 → JSON DomainEvent 서브클래스로 전환
- @JsonTypeInfo 기반 다형성 역직렬화
- 이벤트 클래스 7개: BookingCreated/Confirmed/Cancelled, PaymentCompleted/Refunded, UserCreated, TripCreated

### 9. 서비스 간 통신 (Phase 1D)
- **동기 (gRPC)**: booking → user-service (유저 존재 확인), booking → trip-service (여행 존재 확인)
- **비동기 (Kafka)**: 결제 완료 → 예약 확정, 예약 취소 → 자동 환불
- Proto 모듈 생성 (user_service.proto, trip_service.proto)
- Eureka @LoadBalanced RestClient

### 10. Outbox Pattern
- DB 트랜잭션과 Kafka 발행 일관성 보장
- OutboxEvent 엔티티 → OutboxScheduler가 1초마다 폴링 → Kafka 발행
- 모든 서비스에서 KafkaTemplate 직접 사용 → OutboxEventPublisher로 교체

### 11. XSS 방어
- XssSanitizer: HTML 특수문자 이스케이프 + script 태그 제거
- XssFilter: 모든 /api/* 요청 body/parameter/header sanitize
- SecurityHeaderFilter: X-Content-Type-Options, X-XSS-Protection, X-Frame-Options, CSP

### 12. !! (non-null assertion) 제거
- 모든 서비스에서 `!!` → `requireNotNull()` 또는 `?: throw` 로 교체
- NPE 위험 제거

### 13. Kotlin 전환
- 전체 프로젝트 Java → Kotlin 리팩토링 (~70개 파일)
- build.gradle → build.gradle.kts 전환
- Lombok 완전 제거 (Kotlin data class, val/var 사용)
- kotlin-spring, kotlin-jpa 플러그인 적용
- Kotlin 2.2.21 + Gradle 9.4.1

### 14. QueryDSL + MyBatis 설정
- 각 서비스에 kapt + querydsl-jpa + querydsl-apt
- MyBatis 설정 (mapper-locations, type-aliases-package)
- QueryDslConfig (JPAQueryFactory 빈)

### 15. gRPC 설정
- proto 모듈: protobuf-gradle-plugin + grpc-java + grpc-kotlin
- user-service: UserGrpcServer (GetUser, CheckUserExists)
- trip-service: TripGrpcServer (GetTrip, CheckTripExists)
- booking-service: gRPC 클라이언트로 교체 (REST → gRPC)

### 16. 신규 서비스 6개 구현
- **accommodation-service** (:8085) — 한국관광공사 Tour API 연동, 숙소 검색, 가격 파싱
- **subscription-service** (:8086) — 영상통화 크레딧 (5회 무료, 10회/5,000원), 포트원 결제 검증
- **feed-service** (:8087) — 피드 CRUD (사진+글), S3(MinIO) 이미지 업로드
- **media-service** (:8088) — FFmpeg 트랜스코딩, HLS, Nginx-RTMP 스트리밍
- **chat-service** (:8089) — GPS 기반 채팅방 (Haversine), WebSocket STOMP, Redis 온라인 추적
- **notification-service** (:8090) — SSE 실시간 알림, Kafka 다중 토픽 소비

### 17. Docker Compose 인프라
- MySQL 8.0 (:3000), Redis 7 (:6379), Kafka (:9092), MinIO (:9000), Nginx-RTMP (:1935)
- init SQL 제거 (JPA ddl-auto 사용)

### 18. React 프론트엔드
- Vite + React + TypeScript 프로젝트 셋업
- 16개 페이지 구현 (홈, 로그인, 회원가입, OAuth콜백, 여행CRUD, 예약, 결제, 숙소, 구독, 피드, 채팅, 채팅방, 알림, 스트리밍)
- Redux Toolkit (authSlice + store), React Query, MUI, Styled Components
- API 클라이언트 (axios + JWT 인터셉터 + 자동 리프레시)
- AuthInitializer (앱 진입 시 토큰 검증)
